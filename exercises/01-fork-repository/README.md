# Exercise 1 — Fork Repository

**Skill:** Forking a repository on GitHub

**Time:** ~5 minutes

---

## What is a fork?

A **fork** is your own personal copy of someone else's repository. It lives on your GitHub account and lets you make changes without affecting the original project.

Think of it like photocopying a friend's notes — you can write all over your copy without messing up theirs.

---

## Steps

### 1. Go to the workshop repo on GitHub

Open this link in your browser:

```
https://github.com/Alonzorr10/TUJ-2026-GitHub-Workshop
```

### 2. Click the "Fork" button

- Look at the **top-right** corner of the page
- Click the **Fork** button
- On the "Create a new fork" page, keep all defaults and click **Create fork**

### 3. Wait for the fork to finish

GitHub will redirect you to your brand new copy. The URL will look like:

```
https://github.com/YOUR-USERNAME/TUJ-2026-GitHub-Workshop
```

Notice it says **your username** instead of `Alonzorr10` — that means you own this copy.

### 4. Confirm it worked

Check for these signs that your fork is set up correctly:

- The page title says `YOUR-USERNAME/TUJ-2026-GitHub-Workshop`
- Below the repo name it says **"forked from Alonzorr10/TUJ-2026-GitHub-Workshop"**
- You can see all the same files as the original repo

## Quick Reference

| Term | Meaning |
|------|---------|
| **Upstream** | The original repo you forked from |
| **Origin** | Your personal fork on GitHub |
| **Fork** | A server-side copy of someone else's repo |
